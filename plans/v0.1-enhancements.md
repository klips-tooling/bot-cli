# BOT CLI v0.1 Enhancements Plan

## Context
The core CLI is working end-to-end. Now we want to make the generated projects feel polished and expand the starter options:

1. Give the frontend templates a custom BOT Chain landing page using the brand colors from the media kit (shades of `#1CD8B0` and the SVG logo's `#10A37F`).
2. Add a **Guess the Number** smart-contract starter option.
3. Add the remaining Vite template: **Vite + React**.

## Approach

### 1. BOT-branded landing page
- Build a reusable landing-page design shared across Vanilla, Vue, and React templates.
- Use BOT brand colors:
  - Primary: `#1CD8B0`
  - Dark accent: `#10A37F` (from `BOT Chain.svg`)
  - Background: dark slate gradient (`#0B1120` → `#0F172A`)
- Include:
  - Inline BOT logo SVG (simplified from media kit)
  - "BOT Chain" headline + tagline
  - Feature cards (Fast, Secure, Builder-friendly)
  - Styled "Connect Wallet" button using BOT colors
  - Network badge showing selected Mainnet/Testnet
- Keep both Tailwind and plain-CSS variants:
  - `style.tailwind.css` uses `@import "tailwindcss"` + `@apply` utilities.
  - `style.css` is a hand-written CSS fallback with the same look.

### 2. Guess the Number contract starter
- Move contract-specific files out of `templates/foundry-basic/` into:
  - `templates/contract-starters/counter/`
  - `templates/contract-starters/guess-the-number/`
- Each starter contains `src/`, `test/`, and `script/` files.
- Add a new prompt in `botdev init`:
  - "Choose a contract starter: Counter / Guess the Number"
- After copying the base Foundry template, copy the selected starter into the project.
- Update the Foundry README to use `{{contractName}}` and `{{deployScriptName}}` tokens.

### 3. Vite + React template
- Create `templates/vite-react/` mirroring `vite-vue`:
  - `package.json`, `vite.config.ts`, `tsconfig.json`, `index.html`
  - `src/main.tsx`, `src/App.tsx`
  - `src/botChain.ts`
  - `src/style.css` and `src/style.tailwind.css`
  - `.env.example`, `README.md`, `template.json`
- Use the same BOT landing-page components as Vue/Vanilla.
